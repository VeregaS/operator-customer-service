ТЕХНИЧЕСКОЕ ЗАДАНИЕ
Информационная система: «Обслуживание клиентов оператора сотовой связи»
Язык реализации: Python

1. АРХИТЕКТУРА И СТРУКТУРА ПРОЕКТА
Проект разделен на логические модули:
- Модели данных (классы сущностей).
- Структуры данных (кастомные коллекции).
- Алгоритмы (поиск, сортировка, обход).
- Бизнес-логика (контроллер системы).

2. МОДЕЛИ ДАННЫХ И СТРУКТУРЫ ХРАНЕНИЯ

2.1. Сущность «Клиент» (Client)
- Поля:
  - Номер паспорта (строка, формат "NNNN-NNNNNN") — Первичный ключ.
  - Место и дата выдачи паспорта (строка).
  - ФИО (строка).
  - Год рождения (целое число).
  - Адрес (строка).
- Структура хранения: АВЛ-дерево поиска.
- Ключ сортировки: Номер паспорта.

2.2. Сущность «SIM-карта» (SimCard)
- Поля:
  - Номер SIM-карты (строка, формат "NNN-NNNNNNN") — Первичный ключ.
  - Тариф (строка).
  - Год выпуска (целое число).
  - Признак наличия (логическое: True - у оператора, False - на руках).
- Структура хранения: Хеш-таблица.
- Метод хеширования: Закрытое хеширование с линейным опробованием.

2.3. Сущность «Учетная запись операции» (Record)
- Поля:
  - Номер паспорта (строка).
  - Номер SIM-карты (строка).
  - Дата выдачи (строка).
  - Дата окончания действия (строка).
- Структура хранения: Циклический однонаправленный список.
- Порядок хранения: Упорядочен по ключу «Номер SIM-карты».
- Метод сортировки: Сортировка подсчетом (требует маппинга строковых ключей в целочисленные индексы).

3. АЛГОРИТМЫ

3.1. Поиск в тексте
- Алгоритм: Прямой поиск слова в тексте.
- Применение: Поиск клиентов по фрагментам ФИО или адреса.

3.2. Обход дерева
- Алгоритм: Прямой обход (Pre-order: Корень -> Левое -> Правое).
- Применение: Систематический проход по АВЛ-дереву для поиска по фрагментам ФИО/Адреса.

3.3. Поиск SIM-карты по тарифу
- Алгоритм: Линейный поиск по массиву бакетов хеш-таблицы (сопровождается аналитической запиской со сравнением с альтернативами, например, с вторичным индексом).

4. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ (БИЗНЕС-ЛОГИКА)

4.1. Управление клиентами
- Регистрация нового клиента (вставка в АВЛ-дерево).
- Снятие с обслуживания (удаление из АВЛ-дерева). Блокируется, если у клиента есть невозвращенные SIM-карты.
- Просмотр всех клиентов (вывод минимум ФИО и Номера паспорта).
- Очистка данных о клиентах.
- Точный поиск по паспорту (вывод данных клиента + список номеров его SIM-карт).
- Частичный поиск по ФИО/Адресу (комбинация прямого обхода АВЛ-дерева и прямого поиска подстроки). Вывод: Паспорт, ФИО, Адрес.

4.2. Управление SIM-картами
- Добавление новой SIM-карты (вставка в хеш-таблицу).
- Удаление сведений о SIM-карте. Блокируется, если SIM-карта выдана клиенту (Признак наличия = False). При удалении используется маркер (tombstone) во избежание разрыва цепочек хеш-таблицы.
- Просмотр всех SIM-карт (вывод минимум Номера и Тарифа).
- Очистка данных о SIM-картах.
- Точный поиск по номеру SIM (вывод данных SIM + ФИО и Паспорт текущего владельца, если выдана).
- Поиск по тарифу (вывод Номера, Тарифа, Года выпуска).

4.3. Управление операциями (Выдача/Возврат)
- Регистрация выдачи: 
  - Проверка наличия SIM (Признак наличия == True).
  - Изменение Признака наличия на False.
  - Добавление записи в циклический список.
  - Поддержание сортировки списка.
- Регистрация возврата:
  - Изменение Признака наличия на True.
  - Обновление Даты окончания действия в соответствующей записи циклического списка.

5. ОГРАНИЧЕНИЯ И ТЕХНИЧЕСКИЕ УСЛОВИЯ
- Запрещено использовать встроенные структуры данных Python (dict, list, set) в качестве основы для АВЛ-дерева, Хеш-таблицы и Циклического списка. Все базовые структуры пишутся с нуля на узлах (Nodes) и массивах фиксированного размера (для хеш-таблицы).
- Обработка коллизий в хеш-таблице обязательна.
- Соблюдение принципа DRY (Don't Repeat Yourself) и строгая типизация (Type Hints).
